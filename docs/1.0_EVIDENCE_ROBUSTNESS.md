## A. 决策意义不依赖弱对手（对手非木桩复测）

1.0 主证据里蓝方永远是 `dumb`（木桩）。一个常见质疑：好指挥赢 96%，会不会只是因为对手太弱？这正是《圣兽之王》死掉的病——"深度没被需要，因为敌人 AI 太弱，无脑按推荐就能通关"。

复测：把蓝方换成会打的 `good_focus`，看红方在不同指挥下胜率是否还分得开。

命令样例：

```bash
python3 scripts/mirror_map_sim.py --runs 10000 --jitter 1 --red-policy good_focus --blue-policy good_focus
```

参数：runs=10000, seed=42, jitter=1。

### 头版对照表

| 红方指令 | vs 木桩(dumb) 红胜 | vs 会打的对手(good_focus) 红胜 |
| --- | ---: | ---: |
| 不指挥 (dumb) | 32.6% | **0.0%** |
| 好指挥 (good_focus) | 96.3% | **23.7%** |
| 莽夫 (bad_charge) | 0.3% | 0.2% |

### vs blue=good_focus 全表（红胜 / 蓝胜 / 平）

```text
red=good_focus   23.7% / 23.7% / 52.6%   势均力敌
red=bad_charge    0.2% / 95.7% /  4.0%   莽夫仍灾难
red=cower_all     0.0% /  0.0% /100.0%   双方都不推进 -> 全平
red=hesitate     11.5% / 58.4% / 30.1%   迟疑明显吃亏
red=dumb          0.0% / 96.6% /  3.4%   不指挥被抹平
```

### 对照：vs blue=hold_all（原地开火的对手）

```text
red=good_focus    0.0% /  0.0%   双方龟缩 -> 全平
red=bad_charge    0.0% /100.0%   冲固守阵线必死
red=dumb         18.0% / 51.7%
```

### 结论

对手越强，指挥越被需要，与《圣兽之王》的病**正好相反**。面对会打的对手，"好好指挥"是**全灭(0.0%)与打成五五(23.7%)的区别**；不指挥(dumb)直接被抹平到 0.0%。决策的意义不是木桩制造的幻觉。

### 已知纹理（诚实交底）

双方都打防守向策略时，地图会大量平局：good_focus 互打 52.6% 平、cower_all 对 good_focus 100% 平、good_focus 对 hold_all 100% 平。成因是 16 回合上限 + 无强制交火逼近，属调参纹理，非机制漏洞。列为已知待调项。

## B. LLM 路由抗钓鱼（自然语言 -> policy 现场真调 DeepSeek）

1.0 证据表只留了单次 run 的五行 reason。本轮现场真调 LLM 复测路由稳定性，含故意设陷阱的边界指令。

### 内置矩阵（真调，runs=50）

```text
zero_input       -> dumb        conf 1.00
good_command     -> good_focus  conf 0.95
bad_charge       -> bad_charge  conf 0.95
cower_command    -> cower_all   conf 1.00
irrelevant_chat  -> hesitate    conf 0.90
```

### 陷阱指令（每条真调一次）

```text
"今天适合趴着晒太阳，喝杯咖啡。"            -> hesitate   conf 1.00  （埋"趴"字钓 cower，未咬钩）
"前排冲上去追对面后排，中排原地别动。"      -> bad_charge  conf 0.90  （半好半坏，取主导动作归坏桶）
"辛苦了兄弟们，大家加油打得漂亮！"          -> hesitate   conf 0.90  （鼓励话术未被洗成 good）
"别冲，也别趴下，稳住阵线慢慢磨残血。"      -> good_focus  conf 0.95  （三重否定+正向，抽对）
"敌人太强了，我们还是别打了吧。"            -> cower_all   conf 0.90
"All units rush the enemy backline..."      -> bad_charge  conf 0.95  （英文，跨语言对）
```

### 结论

路由按语义而非关键词工作，抗钓鱼，跨语言成立。唯一边界：6 个桶处理不了"局部正确"的复合指令，只能整句压一个标签——这是 6 分类设计的固有天花板，属下一条延展轴（按角色可组合下令），非 bug。

### 已知纹理

confidence 恒在 0.90~1.00，几乎不波动，未起到区分作用。对 1.0 证据无影响，列为可改进项。
